The full animation workflow from script to finished video. You will understand how to set up the two-workspace folder system, write a spec, have Claude build animations from it, and render the result into something you can edit in CapCut or any video editor.

## The Lesson

**What this actually looks like**

The animations across this channel would have taken a full week to make by hand. Now they take under an hour. And they do not have to look like mine. They can look completely different. Anything Claude can create within a React framework can become an animation.

The workflow is not a single tool. It is a folder system. You write in English, in a document, in a folder. Claude Code follows a series of processes and uses libraries to create the animations. You can use natural language to edit them. You can stay hands-off or you can control every step.

---

## What you need

Four things (two of which you already have from 4.1):

1. **Claude Code** — Already installed from 4.1
2. **Node.js** — Already installed from 4.1
3. **Remotion** — The library that turns React components into video, frame by frame. Claude can install this for you.
4. **A video editor** — CapCut, DaVinci Resolve, or whatever you prefer. For the final edit: voiceover, music, captions, timing adjustments.

---

## Installing Remotion

You can have Claude do this for you. Open Claude Code in a new folder (this will be your animation workspace) and say:

```
Help me install Remotion. Here is the documentation: https://remotion.dev
```

Claude will set up the project. Remotion also has agent skills (a set of files that teach Claude how to use the library). You can install those too:

```
Also install the Remotion agent skills from their GitHub.
```

The skills are just folders full of markdown and code examples. Claude reads them when it needs guidance on 3D content, DOM elements, Tailwind styling, transitions, and more. Claude does not read all of them at once. It reads the parts relevant to the current task.

If you want to install manually instead: go to [remotion.dev](http://remotion.dev/ "http://remotion.dev") for the library and their GitHub for the agent skills. But honestly, letting Claude handle it is faster and it will set up the folder structure correctly.

## The two-workspace setup

I use two folders for this workflow. Both live inside one parent folder.

```
my-animations/
├── CLAUDE.md
├── script-lab/
│   ├── long-form/
│   └── short-form/
└── animation-studio/
    ├── CONTEXT.md
    ├── specs/
    ├── builds/
    └── output/
```

**Script Lab** — Where scripts live. Long-form and short-form in separate subfolders. You write them, or you have Claude write a hundred and pick the best ones. Either way, the finished script ends up here.

**Animation Studio** — Where animations get built. This folder hosts the workflow: specs, builds (the actual React code Claude writes), and output (the rendered video files).

The [CLAUDE.md](http://claude.md/ "http://CLAUDE.md") at the top routes between them. When you say "let's build an animation from a spec," Claude knows to go to the animation studio, read the context file, find the spec, and start building.

If you went through [**3.2: Customizing for Your Use Case**](https://www.skool.com/quantum-quill-lyceum-1116/classroom/036893d9?md=1285fe09df7943e79e5d71c68b8b3ccb&utm_campaign=skool_link_classroom&utm_content=9a42d1d697754c499c3effeee8135e03 "https://www.skool.com/quantum-quill-lyceum-1116/classroom/036893d9?md=1285fe09df7943e79e5d71c68b8b3ccb&utm_campaign=skool_link_classroom&utm_content=9a42d1d697754c499c3effeee8135e03"), this is the content creator structure in action. Script Lab is the writing workspace. Animation Studio is the production workspace.

## The four stages

The workflow has four steps. The AI can automate all four or you can get deeply involved at any step. That is the whole point of having it broken into stages.

#### Stage 1: Script

What needs to be said. Write it yourself, have Claude draft it from a topic, or give Claude a transcript and have it clean it up. The finished script goes in your Script Lab folder.

#### Stage 2: Spec

The specification file. This is the most important file in the entire workflow. It is a markdown document that maps the script to animation beat by beat. Timestamps, visual philosophy, key moments, audio sync points.

The spec is a contract between the voiceover and the animation. It makes sure the timing of the script carries over into when the animations appear.

What a spec contains:

- A beat map (every moment, timestamped)
- Visual philosophy (what the viewer should see and feel)
- Key moments (the beats that need to land hardest)
- Audio sync points (where specific animations need to hit specific words)

What a spec does NOT contain:

- Specific frame numbers
- Pixel positions
- Component props
- Code

I learned this the hard way. Putting code-level detail in the spec actually constrained Claude and made the animations worse. Giving Claude creative room within clear boundaries produces better results. This connects directly to the constraints principle from [**Section 1.3**](https://www.skool.com/quantum-quill-lyceum-1116/classroom/036893d9?md=05230de8023d463f8d38fddc19152ae2&utm_campaign=skool_link_classroom&utm_content=9a42d1d697754c499c3effeee8135e03 "https://www.skool.com/quantum-quill-lyceum-1116/classroom/036893d9?md=05230de8023d463f8d38fddc19152ae2&utm_campaign=skool_link_classroom&utm_content=9a42d1d697754c499c3effeee8135e03") and the inverted-U from [**Section 2.6**](https://www.skool.com/quantum-quill-lyceum-1116/classroom/036893d9?md=3a82e3222c2c4cbc9d07c6f9f6d4265a&utm_campaign=skool_link_classroom&utm_content=9a42d1d697754c499c3effeee8135e03 "https://www.skool.com/quantum-quill-lyceum-1116/classroom/036893d9?md=3a82e3222c2c4cbc9d07c6f9f6d4265a&utm_campaign=skool_link_classroom&utm_content=9a42d1d697754c499c3effeee8135e03"): too few constraints and you get chaos, too many and you get stiff output, the right amount and creativity increases.

You can write the spec yourself, have Claude generate it from the script, or work back and forth. I usually start with Claude generating a first pass, then I edit the parts that matter most.

#### Stage 3: Build

This is where Claude starts writing code. You say "let's build an animation from this spec" and Claude reads the spec, references any documentation or component libraries in the folder, and starts creating React components scene by scene.

Each scene is a separate file. Each file is a React component that Remotion can render frame by frame. You do not need to know React. Claude writes it. You preview it and describe changes in English.

What this looks like in practice: Claude reads the spec. It creates a sequence of scenes. Each scene has animations, timing, transitions. You preview it in Remotion Studio (a browser-based preview that lets you scrub through the animation). If something is not right, you describe the change: "Make that gauge bigger." "Slow down the transition at timestamp 4:30." "I don't like what you changed, go back." Claude adjusts.

The component library matters here. Over time, you build up reusable pieces: text animations, backgrounds, transitions, data visualizations. Claude assembles these instead of inventing everything from scratch each time. This is separation of concerns. The same principle from Section 3 applied to animation production.

#### Stage 4: Render

Ask Claude to render the animation. Remotion compiles all the scenes into a video file. You can also use OBS to screen-record the preview if you want to skip the render process entirely (this avoids codec headaches).

The rendered video goes into your output folder. From there, you bring it into CapCut or your video editor of choice.

---

## The final edit

The animation is not the finished video. It is a strong starting point.

In CapCut (or whatever editor you use), you do the voiceover, sync the audio to the animation, add music, cut sections that do not work, add photos or screenshots or footage, and refine the pacing. Some of this is faster by hand than describing it in a spec. Scrubbing through and nudging a transition to land on a specific word is a video editor task, not an AI task.

When I say the animation takes under an hour, I mean the animation. The finished video still takes time. But that time goes into the parts I actually want to control: pacing, real images, the edit.

---

## How to start your first one today

1. Make sure Claude Code and Remotion are installed in a folder.
2. Write a short script (30 seconds to 1 minute). Or take a transcript from something you have already recorded.
3. Ask Claude to generate a spec from the script: "Here is a script. Write me a spec for a video animation, beat by beat."
4. Review the spec. Edit anything that matters to you.
5. Tell Claude to build it: "Let's build an animation from this spec."
6. Preview in Remotion Studio. Describe changes in English.
7. Render and drop it into your video editor.

Start short. A 30-second animation is a better first project than a 10-minute one. Get the workflow working, then scale up.

Some community members have already been building animations from this process. Share yours in the community. Post what worked and what tripped you up. That is how everyone gets better at this.
