2.2 Pull Structured Data from Any Page

---

## What You'll Get From This

The ability to extract tables, product info, contact details, and any other structured data from websites directly into a format you can use.

---

## The Lesson

## Step 1: Open the Claude Sidebar

Click the Claude icon in your Chrome toolbar. The sidebar opens on the right side of your browser. This is where you'll talk to Claude.

---

## Step 2: Navigate to the Page

Go to the website with the data you want. You can do this yourself (type the URL, use bookmarks, search Google) or you can ask Claude to do it.

To have Claude navigate:

```
Go to amazon.com and search for wireless keyboards
```

Claude will open a new tab, go to Amazon, type in the search, and show you the results page.

---

## Step 3: Tell Claude What You Want

Look at the page. Decide what information matters. Then describe it to Claude.

Example:

```
Pull the product name, price, and star rating for the first 10 results on this page.
```

Claude reads the page, finds those elements, and extracts them.

---

## Step 4: Get the Results

Claude responds with the data formatted as a list or table. Something like:

> "Here are the first 10 results:
> 
> - Logitech K380 - $29.99 - 4.7 stars
> - Apple Magic Keyboard - $99.00 - 4.8 stars
> - Anker Compact Keyboard - $21.99 - 4.4 stars..."

You can copy this directly into a spreadsheet, document, or note.

---

## Step 5: Ask for a Different Format

If you want the data structured differently, just say so:

```
Put that in a table with columns for Name, Price, Rating, and URL
```

Claude reformats the output. You can ask for:

- A table
- A CSV format (for direct spreadsheet import)
- A bulleted list
- A numbered list
- JSON (if you're a developer)

---

## Working with Multiple Pages

Most websites spread data across many pages. Pagination. Scroll-to-load. Individual detail pages. Claude handles these.

**For pagination (Next buttons):**

```
Go through the first 5 pages of results and pull the same information from each page.
```

Claude clicks to the next page, extracts, repeats.

**For scroll-to-load pages:**

```
Scroll down to load more results, then pull the data.
```

Claude scrolls, waits for content to load, then extracts.

**For detail pages:**

```
Click into each of the first 5 listings and pull the full description and seller info.
```

Claude opens each page, grabs the details, and compiles everything.

## A Concrete Example to Practice

Try this with a real site:

1. Open Chrome and go to [https://books.toscrape.com](https://books.toscrape.com/ "https://books.toscrape.com") (a practice site for data extraction)
2. Open the Claude sidebar
3. Type:
```
Pull the title, price, and star rating for every book on this page.
```
4. Claude extracts and lists them
5. Now try:
```
Click to page 2 and do the same thing.
```
6. Now try:
```
Give me all of that as a CSV I can paste into a spreadsheet.
```

## What Claude Can Read

Claude reads what's visible on the page. This includes:

- Text content
- Prices
- Ratings
- Descriptions
- Links
- Tables
- Lists

Claude can also read text inside images if it's clear enough.

---

## What Claude Can't Read

- **Content behind login walls** (Claude will ask you to log in manually first)
- **Content that loads only after you interact** (dropdowns, hover menus)
- **Data hidden until you scroll** (tell Claude to scroll first)
- **PDFs and downloads** (Claude reads web pages, not files)

---

## Working with Multiple Tabs

Claude can work across tabs at once. Drag tabs into Claude's tab group (it creates one when you start working), and Claude can see all of them.

Example:

1. Open three competitor websites in separate tabs
2. Drag all three into Claude's tab group
3. Type:
```
Look at all three tabs and pull the pricing for their basic plan.
```

Claude checks each tab and compiles a comparison.

---

## Saving Time on Repeat Tasks

If you do the same extraction regularly (checking competitor prices every week, pulling new job listings), you can save the prompt as a shortcut:

1. Get your prompt working the way you want
2. Click the settings icon in the Claude sidebar
3. Save the prompt as a shortcut
4. Next time, just type `/` and select your shortcut

## Common Problems

**"Claude pulled the wrong information"** Be more specific. Instead of "get the prices," say "get the price shown in bold next to each product title." Point Claude to a specific area: "Look at the product grid on the left side of the page."

**"Claude missed some items"** The page might need to scroll. Say "scroll down to load all items first, then extract." Some content might be hidden behind "Show More" buttons. Tell Claude to click those first.

**"The site looks different to Claude"** Some sites show different layouts to different users. Ask Claude to describe what it sees. Pop-ups or modals might be blocking the view. Tell Claude to close them.

**"I can't extract from this site"** Some sites block automated access. Try a different site. If it's behind a login, log in manually first, then try again.

**"The data is messy"** Ask Claude to clean it up: "Remove any items without a price" or "Only include items that have all three fields."

---

## Tips for Better Extractions

- **Be specific about what you want.** "Get the product info" is vague. "Get the product name, price, and number of reviews" is clear.
- **Tell Claude where to look.** "In the main product grid" or "from the table in the middle of the page" helps Claude find the right content.
- **Start small.** Extract from one page first. Make sure it works. Then scale to multiple pages.
- **Ask Claude to verify.** "How many items did you find?" catches missing data early.
- **Request the format you need.** Ask for CSV if it's going into a spreadsheet. Ask for a table if you want to read it in the chat.

## Resources

**Example Prompts for Common Extractions**

These show the prompting framework from Lesson 1.3 applied to browser tasks. Copy and adapt for your own use.

*Product research:*

```
Go to [competitor site]. Find their pricing page. Pull the plan names, prices, and what's included in each tier. Format as a table.
```

*Contact scraping:*

```
Look at this company's About or Team page. Pull every person's name, title, and email address if visible. Skip anyone without an email listed.
```

*Job listings:*

```
Go to [job board] and search for [role] in [location]. Pull the job title, company name, salary if shown, and the link to each posting for the first 20 results. Format as CSV.
```

*Review aggregation:*

```
Look at the reviews on this product page. Pull the star rating, review title, and first sentence of each review for the 10 most recent reviews.
```

*Price monitoring:*

```
Check these three tabs. Pull the product name and current price from each. Tell me which one is cheapest.
```
