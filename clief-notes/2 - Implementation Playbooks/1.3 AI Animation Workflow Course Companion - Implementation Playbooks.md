## Quick Reference

#### The Four Stages

Stage What It Is Input Output **Script** What needs to be said Your ideas, notes, transcript Plain text script **Spec** Contract between voice and visuals Script Markdown file with beats, philosophy, key moments **Build** Claude writes the code Spec React components, scenes, animation files **Render** Remotion outputs video Built project MP4 or MOV file

---

## Folder Structure

```
Animation Studio/
├── CLAUDE.md              # Context file - tells Claude where everything is
├── scripts/               # Input scripts (plain text or markdown)
│   ├── long-form/
│   └── short-form/
├── specs/                 # Specification files
│   └── [project-name].md
├── projects/              # Built animation projects
│   └── [project-name]/
│       ├── src/
│       │   ├── compositions/
│       │   ├── components/
│       │   └── scenes/
│       └── remotion.config.ts
└── output/                # Rendered videos
    └── [project-name].mp4
```

You don't need to create all of this manually. Claude builds it as you work. But knowing the structure helps when you need to find something.

## Essential Commands

**Install Remotion in a new folder:**

```
Help me install Remotion. Here's the GitHub: https://github.com/remotion-dev/remotion
I also need Node.js if I don't have it.
```

**Generate a spec from a script:**

```
Here's my script: [paste or point to file]
Write a spec for this. I want [describe visual style].
```

**Build from a spec:**

```
Let's build the animation from this spec.
```

**Preview the animation:**

```
Start the Remotion preview so I can see the animation.
```

**Render to video:**

```
Render this animation to a video file.
```

**Edit something specific:**

```
Make the transition at [timestamp] slower.
Change the accent color to [color].
The text at [timestamp] is hard to read. Make it bolder.
```

## What Goes in a Spec

**Include:**

- Beat map (what happens at each timestamp)
- Visual philosophy (colors, style, mood)
- Key moments (what needs emphasis)
- Audio sync points (where animations should hit)

**Don't include:**

- Frame numbers
- Pixel positions
- Component props
- Code

The spec is creative direction, not technical instruction. Give Claude room to make decisions.

## SVG Export Settings (Illustrator)

Setting Value Format SVG Styling Internal CSS Object IDs Layer Names Minify Off Responsive On

The critical one: **Object IDs: Layer Names**. This preserves your layer naming so Claude can find elements.

## Layer Naming Conventions

**For characters:**

```
[character]_[part]_[side]

hero_eye_left
hero_eye_right
hero_eyebrow_left
hero_mouth
hero_head
```

**For scenes:**

```
[layer]_[element]_[number]

bg_mountain_01
fg_tree_left
mid_cloud_03
```

## Troubleshooting

**"Claude can't install Remotion"** Make sure Node.js is installed first. Download from [https://nodejs.org](https://nodejs.org/ "https://nodejs.org")

**"Render failed"** Check that all your imports are correct. Ask Claude: "Are there any errors in the project?"

**"Preview won't start"** Another process might be using the port. Ask Claude to kill it and restart.

**"Animation timing is off"** Your spec timestamps might not match your audio. Review the beat map.

**"Claude made changes I didn't want"** Be more specific about what to change. "Edit the eyes" is vague. "Slow down the blink speed on the hero character" is precise.

**"SVG layers aren't showing up correctly"** Re-export from Illustrator with Object IDs set to "Layer Names."

**"The file structure is a mess"** Ask Claude to reorganize: "Clean up this project folder and update [CLAUDE.md](http://claude.md/ "http://CLAUDE.md") to reflect the structure."

## The Workflow in 60 Seconds

1. Write or paste your script
2. Ask Claude to generate a spec (describe your visual style)
3. Review the spec, edit what you don't like
4. Tell Claude to build it
5. Preview and make natural language edits
6. Render to video
7. Drop into CapCut/Premiere, add voiceover, export

That's it. Script → spec → build → render.
