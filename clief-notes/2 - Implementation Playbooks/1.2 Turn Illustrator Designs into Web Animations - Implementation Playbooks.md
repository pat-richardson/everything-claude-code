## What You'll Get From This

A workflow for taking hand-designed assets from Adobe Illustrator and turning them into animated, editable components you control with natural language.

---

## The Lesson

#### What This Is For

[**Lesson 1.1**](https://www.skool.com/quantum-quill-lyceum-1116/classroom/d3907117?md=f7a33a9888604a08a7e48bb876682691&utm_campaign=skool_link_classroom&utm_content=9a42d1d697754c499c3effeee8135e03 "https://www.skool.com/quantum-quill-lyceum-1116/classroom/d3907117?md=f7a33a9888604a08a7e48bb876682691&utm_campaign=skool_link_classroom&utm_content=9a42d1d697754c499c3effeee8135e03") showed you how to build animations from scratch using specs. This lesson is different.

Here, you start with designs that already exist. Characters you drew. Storyboards you created. Assets from Illustrator, Figma, or any design tool that exports SVG.

The goal: take those static designs and make them move. Eyes that blink. Expressions that shift. Elements that animate on cue. All editable with plain English.

---

#### Why SVG

SVG files are basically HTML. They're text files that describe shapes, colors, positions. Claude can read them. Claude can edit them. Claude can animate them.

When you export from Illustrator as SVG, you get a file that looks like code. That's the point. It becomes something Claude can work with directly.

Other formats (PNG, JPG) are flat images. Claude can display them but can't edit individual elements. SVG keeps everything separate and editable.

## Step 1: Label Your Layers in Illustrator

This is the most important prep step. Before you export, name your layers so Claude knows what's what.

**Bad layer names:**

- Layer 1
- Layer 2
- Group 3
- Path 47

**Good layer names:**

- left\_eye
- right\_eye
- left\_iris
- right\_iris
- eyebrow\_left
- eyebrow\_right
- mouth
- head\_outline

Claude reads these names to understand the structure. When you say "make the eyes blink," Claude looks for layers with "eye" in the name. If your layers are called "Layer 1" and "Group 3," Claude has to guess. It guesses wrong.

The more specific your labels, the better Claude's edits.

For characters with expressions:

- Label each facial feature separately
- Group related elements (all parts of the left eye in one group called "left\_eye")
- Use consistent naming across multiple characters

For scenes with multiple elements:

- Label foreground vs background
- Name interactive elements clearly
- Group things that move together

---

## Step 2: Export as SVG

In Illustrator: File → Export → Export As → SVG

**Export settings that matter:**

- **Styling**: Internal CSS or Inline Style (either works, Internal CSS is cleaner)
- **Object IDs**: Layer Names (this preserves your labels)
- **Minify**: Off (Claude reads it better unminified)
- **Responsive**: On

The key setting is **Object IDs: Layer Names**. This makes sure your careful labeling survives the export.

---

## Step 3: Give Claude the SVG

Put the SVG file in your Animation Studio folder (or whatever you're calling your workspace).

Open Claude Code and point it at the file:

```
Look at character_01.svg. I want to animate this character. The eyes should blink, the eyebrows should move with emotion, and the mouth should have different states for different expressions.
```

Claude reads the SVG, finds your labeled layers, and starts building components.

---

## What Claude Builds

From one SVG, Claude creates:

**Character components** - A React component for the whole character that accepts props like "emotion" and "blinkSpeed."

**Animation profiles** - Definitions for different states. Joy has raised eyebrows and wide eyes. Sadness has drooped features. Fear has rapid blinking.

**Timing controls** - Variables for how fast things move. Iris speed. Blink duration. Transition smoothness.

**Scene integration** - Code that puts the character into a Remotion scene so it can be rendered to video.

All of this stays in separate files. The character definition is one file. The emotion profiles are another. The scene composition is another. Everything is editable independently.

---

## Step 4: Preview and Edit

Run the Remotion preview to see your animation:

```
Start the Remotion preview so I can see the character animation.
```

Claude starts a local preview. You watch it in your browser.

Don't like something? Say so:

```
The eyes are blinking too fast. Slow it down.
```
```
I don't like the sadness expression. The eyebrows should droop more.
```
```
Can we add a subtle head tilt when the character is curious?
```

Claude finds the relevant component, makes the change, and the preview updates.

## Why This Works

Remember the folder architecture from the Foundation course. Your workspace has context at the top ([CLAUDE.md](http://claude.md/ "http://CLAUDE.md"), project files) that tells Claude where everything is.

When you say "I don't like the eyes on sadness," Claude doesn't read the entire codebase looking for eye code. It checks the context file, finds the path to emotion profiles, goes directly there, and edits just that piece.

This is surgical editing. Change one thing without breaking everything else.

Compare this to AI image generation. If you generate an image and don't like the eyes, you regenerate the whole image. Maybe you get better eyes. Maybe you lose the parts you liked. It's a slot machine.

With this workflow, you keep what works and fix what doesn't. The folder structure makes that possible.

## Multiple Characters, Same Workflow

If you have multiple characters:

1. Label each character's layers with a prefix: `char1_left_eye`, `char2_left_eye`
2. Export each character as a separate SVG, or export the full scene with clear grouping
3. Claude builds separate components for each character
4. Each character has its own emotion profiles and timing controls

You can then compose scenes with multiple characters:

```
Create a scene where character 1 looks happy and character 2 looks skeptical. They should be side by side.
```

Claude pulls both components and puts them in the same scene.

## From Animation to Website

The same components work outside of Remotion.

Once you have an animated character component, you can drop it into a React website. The character animates on scroll, responds to hover, reacts to user interaction.

This is what transcript 2 in the video shows: a storyboard website where characters have micro-expressions, react to each other, and animate as you scroll through the content.

The Remotion workflow builds the components. Those components work anywhere React works.

## Common Problems

**"Claude can't find my layers"** Your layer names didn't survive export. Check that Object IDs is set to "Layer Names" in the SVG export settings.

**"The animation looks choppy"** Timing values might be off. Ask Claude to smooth the transitions or slow down specific movements.

**"Claude broke something when it edited"** This usually means the context file doesn't point to the right locations. Make sure your [CLAUDE.md](http://claude.md/ "http://CLAUDE.md") maps where character files, emotion profiles, and scenes live.

**"The SVG file is huge"** Illustrator sometimes exports unnecessary data. Ask Claude to clean up the SVG before animating it.

**"Colors look different in the animation"** SVG color formats vary. Some use hex, some use RGB. Ask Claude to standardize the color definitions.

---

## The Bigger Picture

This is appless work.

You designed in Illustrator. You're animating with Claude Code. You're previewing in a browser. You're editing with natural language.

The folder structure is the software. Claude Code is the runtime. No animation app required.

This is the abstraction shift the Foundation course talks about. The design tool outputs a format (SVG) that becomes input for the next layer (Claude + Remotion). Each tool does what it's good at. The folder holds it all together.

**Layer Naming Conventions**

For characters:

```
[character]_[part]_[side]
Examples:
- hero_eye_left
- hero_eye_right  
- hero_eyebrow_left
- hero_mouth
- hero_head
```

For scenes:

```
[layer]_[element]_[number]
Examples:
- bg_mountain_01
- fg_tree_left
- mid_cloud_03
```
