Production CLAUDE.md Examples

## What You'll Get From This

Five complete, production-grade [CLAUDE.md](http://claude.md/ "http://CLAUDE.md") files for five different use cases. Pick the closest one to your work and edit it.

---

## What This Is

In [The Foundation (4.4)](https://www.skool.com/quantum-quill-lyceum-1116/classroom/036893d9?md=f0655889bd8c49b0a99f5afb9eb03229&utm_campaign=skool_link_classroom&utm_content=9a42d1d697754c499c3effeee8135e03 "https://www.skool.com/quantum-quill-lyceum-1116/classroom/036893d9?md=f0655889bd8c49b0a99f5afb9eb03229&utm_campaign=skool_link_classroom&utm_content=9a42d1d697754c499c3effeee8135e03"), you wrote a basic [CLAUDE.md](http://claude.md/ "http://CLAUDE.md"). These are the advanced versions. Five real setups, each with full routing tables, naming conventions, workspace descriptions, and rules.

## The five examples:

#### 🟠 Solo Content Creator

Multiple workspaces for scripting, production, and distribution. Voice and style rules baked in. Naming conventions for every content stage.

#### 🟡 Freelance Consultant

Multi-client setup with strict confidentiality rules. Client template folder you duplicate for each new engagement. Templates workspace. Business dev workspace with pipeline stages.

#### 🟢 Software Developer

Planning, src, docs, and ops workspaces. Tech stack, commands, conventions, and avoid-list. Skills column in the routing table.

#### 🔵 Researcher / Academic Writer

Sources, analysis, writing, and admin workspaces. Citation standards, key sources list, and current project status tracker.

#### 🟣 Small Business / Non-Technical

Communications, planning, clients, and resources workspaces. Brand voice rules, team info, and operational context. No code required.

## How to use them

Read through all five. Find the closest match. Copy it into a new file called [CLAUDE.md](http://claude.md/ "http://CLAUDE.md"). Edit every bracketed field with your actual information. Delete sections that do not apply. Add anything specific to your work. Save it in the root of your project folder.

These are more detailed than the starter version from the free course. The difference is production-grade routing tables, real naming conventions, and context that has been tested across actual workflows.
