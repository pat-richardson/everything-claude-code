## What You'll Get From This

A complete reference for structuring folders for AI workflows. The three layers, routing tables, naming conventions, and file trees for four different use cases.

---

## What This Is

This is the reference companion to the Folder Architecture lessons in The Foundation Module ([**Section 3**](https://www.skool.com/quantum-quill-lyceum-1116/classroom/036893d9?md=2b4a8ab7461c4f6d828e21c0eb196a6a&utm_campaign=skool_link_classroom&utm_content=9a42d1d697754c499c3effeee8135e03 "https://www.skool.com/quantum-quill-lyceum-1116/classroom/036893d9?md=2b4a8ab7461c4f6d828e21c0eb196a6a&utm_campaign=skool_link_classroom&utm_content=9a42d1d697754c499c3effeee8135e03")). It covers:

- The three-layer system (map, rooms, tools) and what goes in each
- How to build and use routing tables
- Naming conventions that replace databases
- File trees for four use cases: content creator, freelancer/consultant, developer, and researcher
- The seven common mistakes and how to avoid them

You can read it as a reference, hand it to Claude to design a custom folder structure for your work, or print it and keep it next to your screen.

**Pro tip:** Upload this document into a Claude conversation and say: "Read the Folder Organization Guide. I am a \[your role\]. Help me design my folder structure based on what I do." Claude will use the examples and principles to build something custom for you.
