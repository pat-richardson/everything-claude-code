## What You'll Get From This

A curated list of open-source AI UI projects worth studying, with notes on what to learn from each.

## The Lesson

## How to Use This Page

This isn't a tutorial. It's a research resource.

Each repo below does something interesting with AI interfaces. Some wrap Claude Code. Some build custom chat UIs. Some solve problems you haven't thought of yet.

For each one, I've noted:

- What it does
- What's worth studying
- What you might borrow for your own builds

You don't need to clone all of these. Pick the ones relevant to what you're building. Read their code. See how they solved problems. Take what's useful.

This page gets updated as I find new repos worth studying.

---

## Claude Code Interfaces

### Claude Code Web

**What it does:** A web-based front-end for Claude Code. Lets you run Claude Code through a browser interface instead of the terminal.

**What's worth studying:**

- How they handle session persistence
- The websocket connection between front-end and Claude Code CLI
- Their approach to streaming responses

**What I borrowed:** The session persistence patterns. Being able to close the browser and come back to the same conversation.

---

### Op Code

**What it does:** Another Claude Code wrapper with a focus on project management. Tracks usage, manages multiple projects, provides a cleaner interface than raw terminal.

**What's worth studying:**

- Usage tracking implementation
- Multi-project switching
- How they structured the settings/config system

**What I didn't use:** Most of the UI complexity. Too many features for what I needed. But the underlying patterns are solid.

---

### Claude CLI UI

**What it does:** A terminal UI (TUI) for Claude Code. Still in the terminal, but with better visual organization.

**What's worth studying:**

- How they parse and display Claude's output
- Keyboard shortcuts and navigation patterns
- The balance between terminal simplicity and added features

**Good for:** People who want to stay in the terminal but want a better experience than raw CLI.

---

## Custom AI Chat Interfaces

### Open WebUI

**What it does:** A self-hosted web interface for various LLMs. Supports multiple providers, local models, and has a clean chat UI.

**What's worth studying:**

- Multi-provider architecture (how they abstract different AI backends)
- The conversation management system
- Their approach to system prompts and presets

**What you might borrow:** If you're building something that needs to work with multiple AI providers, study how they did the abstraction layer.

---

### Lobe Chat

**What it does:** A polished open-source chat UI with plugins, agents, and knowledge base features.

**What's worth studying:**

- The plugin architecture
- How they handle file uploads and knowledge bases
- UI/UX patterns for chat interfaces

**Good for:** Inspiration on what a full-featured chat interface can look like. Even if you're building something minimal, seeing the full-featured version helps you understand what to cut.

---

## Workflow and Agent Tools

### n8n

**What it does:** A workflow automation tool with a visual node-based editor. Not AI-specific, but increasingly used for AI workflows.

**What's worth studying:**

- The node-based visual editor implementation
- How they handle workflow state and execution
- Their approach to connecting different services

**Why it's here:** The workflow map I want to build for my front-end is conceptually similar. Visual representation of processes. Nodes you can click into. This is one way to solve that problem.

---

### Langflow

**What it does:** A visual framework for building LLM applications. Drag-and-drop interface for creating AI workflows.

**What's worth studying:**

- How they visualize LLM chains and agents
- The component system (each node is a component)
- How they handle the gap between visual representation and actual execution

**Relevant to:** Anyone building visual interfaces for AI workflows. Even if you don't use their approach, understanding it helps you design your own.

---

## Design and UI Skills

### UIUX Pro Skill

**What it does:** A skill package that teaches Claude better design sense for front-end work. Not a repo you run, but prompts and instructions Claude reads.

**What's worth studying:**

- How they structured the skill (prompts, examples, guidelines)
- The design principles they encode
- How skills can shape Claude's output style

**Why it matters:** If you're building a front-end with Claude, this skill makes the output look better. It's also a good example of how to create your own skills for specific domains.

---

## What to Do With These

**If you're building a Claude Code front-end:** Start with Claude Code Web and Op Code. Understand how they connect to the CLI. Borrow the session persistence patterns.

**If you're building a custom chat interface:** Look at Open WebUI and Lobe Chat. Even if they're more complex than what you need, they show patterns for conversation management, streaming, and UI.

**If you're building workflow visualization:** Study n8n and Langflow. The node-based visual editor is a solved problem. See how they solved it.

**If you want Claude to build better front-ends:** Get the UIUX Pro Skill. It's not code to run, it's context for Claude.
